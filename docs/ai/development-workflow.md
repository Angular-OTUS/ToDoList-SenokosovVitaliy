# Development Workflow

## Starting the app locally

```bash
npm run api   # json-server db.json --port 3000  (Terminal 1)
npm start     # nx serve → localhost:4200         (Terminal 2)
```

Both must run. API missing → `catchError` returns `[]`, `isLoading` sets false, task list shows empty (no spinner stuck). Angular missing → blank page.

## Component generator

`component.ps1` in the repo root generates a component via Nx:

```powershell
.\component.ps1 MyComponent
# Runs: nx g @nx/angular:component src/components/my-component/my-component.ts
```

- Converts `PascalCase` or arbitrary names to `kebab-case` automatically.
- Generates `.ts`, `.html`, `.css` files in `src/components/<kebab-name>/`.
- Does NOT add `changeDetection: ChangeDetectionStrategy.OnPush` — add it manually.
- Does NOT add the component to any module (standalone components need no module registration).

**Source:** `component.ps1:1–24`

## CI pipeline

**File:** `.github/workflows/ci.yml`

Triggers on: push to `master`, all pull requests.

Steps:
1. Checkout with full history (`fetch-depth: 0` for Nx affected commands)
2. Setup Node 20 with npm cache
3. `npm ci --legacy-peer-deps`
4. `npx playwright install --with-deps`
5. `npx nx run-many -t lint test build e2e`

Note: `npm ci --legacy-peer-deps` — legacy peer deps flag is required. Do not change package installs without verifying this still works.

Note: CI triggers on `master`, but the main branch in this repo is `main`. PRs to `main` will trigger the workflow via the `pull_request` event. **Source:** `.github/workflows/ci.yml:4`
